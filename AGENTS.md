# Shipwrecked Pools App — Codex Instructions

This is the initial temporary repo-level instruction file.

Sprint 00 will replace and expand this file.

## Project mission

Build a premium true mobile app and internal operating system for Shipwrecked Pools.

The app should eventually replace Skimmer for:
- customer communication
- route management
- service reports
- service photos
- chemistry readings
- billing
- customer records
- repair/quote approvals
- technician workflows
- internal CRM

## Important product priorities

1. Customer experience matters most.
2. The custom top-down pool outline is a critical V1 feature.
3. Customers should see customer-friendly notes only.
4. Technicians should never see billing, payment, route profitability, customer profitability, or sensitive business financial data.
5. Gate codes, pet notes, customer phone numbers, and access reminders may be visible to assigned technicians.
6. Questions in V1 must be answered by humans, not AI.
7. Reports should be generated from technician inputs.
8. Quote approvals must include approval action, checkbox, typed signature, audit log, and payment flow.
9. Route progress should show stops-before-you and ETA, not exact live GPS.
10. All sensitive actions should be auditable.

## Development expectations

- Keep changes small and scoped.
- Do not make unrelated changes.
- Update docs when implementation changes behavior.
- Add tests when building logic.
- Create handoff notes after each prompt pack.
- Respect the prompt pack sequence and dependencies.
